# Soupify Format Instructions (v0.2.0)

You will receive a "soup" file — a markdown file bundling multiple source files. Parse it, do the work, and return results in soup format so the user can restore files locally.

## Document Structure

Zero or more **meta blocks** (metadata, skip on desoupify) followed by zero or more **file blocks**. Blocks concatenate directly — no separators. A trailing newline after the last block is tolerated.

## File Block Header

```
#SOUP "<json_path>" [#SOUP_PARTIAL_LINES <s>-<e>] #SOUPED_LINES <N> #SOUP_TRAILING_NEWLINE <0|1>
```

- **Path**: Absolute path, JSON-encoded (backslashes → `\\`, quotes → `\"`). Preserve exactly.
- **#SOUP_PARTIAL_LINES** *(optional)*: Replace only lines `s`–`e` (1-indexed, inclusive). Rest unchanged.
- **#SOUPED_LINES**: Exact content line count after this header.
- **#SOUP_TRAILING_NEWLINE**: `1` = file ends with `\n`, `0` = it does not.

## Meta Block Header

```
#SOUP_META "<label>" #SOUP_META_KIND <kind> #SOUP_META_FORMAT <fmt> #SOUP_META_LINES <N> #SOUP_META_READONLY <true|false>
```

Generated by `soupify -g`. Parse N lines of content then **skip** — do not write to disk. Do not emit these yourself.

## Parsing (Desoupify)

1. Scan for `#SOUP_META` or `#SOUP` header lines.
2. **Meta block**: consume `#SOUP_META_LINES N` lines, discard.
3. **File block**: extract path (JSON-decode), optional partial range, `N`, trailing-newline flag; consume exactly N lines.
4. Reconstruct file:
   - **Full**: join lines with `\n`, append `\n` if flag is `1`. Write to disk.
   - **Partial**: read target file (post any prior partial edits to it), replace lines `s`–`e`, write back.

## Outputting Edits (Resoupify)

Only emit files you modified or created. Two options per file:

**Full block** — rewritten or new file:
1. Strip trailing `\n`, split on `\n`, count segments (empty → 0).
2. `#SOUP "<path>" #SOUPED_LINES <N> #SOUP_TRAILING_NEWLINE <0|1>` then content.

**Partial block** — small change in a large file:
1. `#SOUP "<path>" #SOUP_PARTIAL_LINES <s>-<e> #SOUPED_LINES <N> #SOUP_TRAILING_NEWLINE <0|1>` then replacement lines only.
2. Trailing-newline flag = state of the **whole file** after the edit.
3. When multiple partial blocks target the same file, line numbers in later blocks are relative to the file **after** all prior partials.

## Line Counting

| Content | Lines | Trailing |
|---|---|---|
| *(empty)* | 0 | 0 |
| `hello` | 1 | 0 |
| `hello\n` | 1 | 1 |
| `a\nb\n` | 2 | 1 |
| `a\nb` | 2 | 0 |
| `one\n\nthree\n` | 3 | 1 |

## Examples

**Two-file soup:**
```
#SOUP "/home/user/src/main.rs" #SOUPED_LINES 3 #SOUP_TRAILING_NEWLINE 1
fn main() {
    println!("Hello!");
}
#SOUP "/home/user/Cargo.toml" #SOUPED_LINES 2 #SOUP_TRAILING_NEWLINE 0
[package]
name = "hello"
```

**With meta block** (from `soupify -g`) — meta content is skipped, only `main.rs` written:
```
#SOUP_META "repo-graph" #SOUP_META_KIND codegraph #SOUP_META_FORMAT repomap #SOUP_META_LINES 1 #SOUP_META_READONLY true
<graph data>
#SOUP "/home/user/src/main.rs" #SOUPED_LINES 1 #SOUP_TRAILING_NEWLINE 1
fn main() {}
```

**Partial edit** — replace lines 2–3, file has trailing newline:
```
#SOUP "/home/user/file.txt" #SOUP_PARTIAL_LINES 2-3 #SOUPED_LINES 2 #SOUP_TRAILING_NEWLINE 1
dos
tres
```

## Critical Rules

- Preserve absolute paths exactly; JSON-escape them.
- Line counts must be exact — off-by-one corrupts the file.
- Serialize trailing newline as `0` or `1`, never `true`/`false`.
- Empty files: `#SOUPED_LINES 0 #SOUP_TRAILING_NEWLINE 0`, no content lines.
- Do not emit `#SOUP_META` blocks.
- Prefer partial blocks for large files with small changes.
