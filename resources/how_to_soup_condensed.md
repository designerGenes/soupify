When outputting code edits, use Soupify v0.3.0. Output ONLY modified/new files as a SINGLE fenced `soup` code block. Blocks concatenate with NO blank lines. FILE HEADER: #SOUP "<json_escaped_abs_path>" [#SOUP_PARTIAL_LINES <s>-<e>] #SOUPED_LINES <N> #SOUP_TRAILING_NEWLINE <0|1> [#SOUP_BASE_SHA <hex64>] [#SOUP_READONLY true]. META (soupify -g): #SOUP_META ... #SOUP_META_LINES <N> ... — parse N lines, skip, do NOT emit. Path: absolute, EXACT, JSON-escaped. #SOUP_PARTIAL_LINES: optional, s-e 1-indexed inclusive; need full file content first or ask. #SOUPED_LINES: exact content line count. #SOUP_TRAILING_NEWLINE <0|1>. #SOUP_BASE_SHA: carry forward on partial returns so host detects drift. #SOUP_READONLY true: context-only, never modify or resoupify. LINE COUNTING: strip trailing \n, split on \n, count; empty=0. OUTPUT: >=50% changed→FULL (emit all N lines); else PARTIAL (emit N replacement lines; trailing-newline=FINAL whole-file state; multiple partials same file→ASCENDING start_line, later lines relative to post-prior-edit state; if risky emit FULL). NO blank lines between header/content/blocks. If map references a file not in soup, emit `#SOUP_REQUEST "<path>" <reason>` in text — do NOT hallucinate contents.
