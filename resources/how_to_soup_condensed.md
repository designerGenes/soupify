When outputting code edits, use Soupify v0.3.0. Output ONLY modified/new files as a SINGLE fenced `soup` code block (avoids #SOUP header misrender). Blocks concatenate with NO blank lines between them. FILE HEADER: #SOUP "<json_escaped_abs_path>" [#SOUP_PARTIAL_LINES <start>-<end>] #SOUPED_LINES <N> #SOUP_TRAILING_NEWLINE <0|1>. META HEADER (soupify -g / --include-graph): #SOUP_META "<label>" #SOUP_META_KIND <kind> #SOUP_META_FORMAT <fmt> #SOUP_META_LINES <N> #SOUP_META_READONLY <true|false> — parse N lines, skip, do NOT emit. Path: absolute, EXACT, JSON-escaped. #SOUP_PARTIAL_LINES: Optional, <start>-<end> 1-indexed inclusive; target file's full content must be known (earlier full block, prior soup, or disk) — else ask user for a full block. #SOUPED_LINES: exact count of content lines following header. #SOUP_TRAILING_NEWLINE <0|1>: 1 if entire file ends in \n. LINE COUNTING: strip trailing \n, split on \n, count segments; empty=0. OUTPUT: replacement lines >= 50% of file → FULL (omit #SOUP_PARTIAL_LINES, emit all N); else PARTIAL (emit N replacement lines only; trailing-newline = FINAL whole-file state; multiple partials same file → ASCENDING start_line, later line numbers relative to file after prior partial edits; if line-shift risky, emit one FULL block). NO blank lines between header/content or blocks. #SOUP_AUTO_DESOUPIFY: not an in-file tag, host-side config only.
