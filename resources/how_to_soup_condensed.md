When outputting code edits, use the Soupify format (v0.2.0). Output ONLY modified/new files. Prefer PARTIAL blocks for large files. Blocks must concatenate directly with NO blank lines between them. 1. FILE BLOCK HEADER: #SOUP "<json_escaped_abs_path>" [#SOUP_PARTIAL_LINES <start>-<end>] #SOUPED_LINES <N> #SOUP_TRAILING_NEWLINE <0|1> 2. META BLOCK (from soupify -g): #SOUP_META "<label>" #SOUP_META_KIND <kind> #SOUP_META_FORMAT <fmt> #SOUP_META_LINES <N> #SOUP_META_READONLY <true|false> — parse N lines then skip, do NOT write to disk, do NOT emit. 3. HEADER PARAMETERS: Path: Original absolute path EXACTLY, JSON-escaped. #SOUP_PARTIAL_LINES: Optional. <start>-<end> are 1-indexed inclusive line numbers being replaced. #SOUPED_LINES: Exact integer count of content lines immediately following the header. #SOUP_TRAILING_NEWLINE <0|1>: 1 if entire file ends in \n, else 0; always 0 or 1. 4. LINE COUNTING: Empty=0. foo=1. foo\n=1. foo\nbar=2. foo\nbar\n=2. CRITICAL: counts must be exact. 5. OUTPUT RULES: FULL BLOCK: omit #SOUP_PARTIAL_LINES, emit all N lines. PARTIAL BLOCK: emit N replacement lines only; trailing-newline flag = FINAL whole-file state; multiple partials on same file — later line numbers are relative to file state after prior partial edits. SPACING: zero blank lines between header and content or between blocks.
