When outputting code edits, use the Soupify format. Output ONLY modified/new files. Prefer PARTIAL blocks for large files. Blocks must concatenate directly with NO blank lines between them. 1. HEADER SYNTAX (Mandatory per block): #SOUP "<json_escaped_abs_path>" [#SOUP_PARTIAL_LINES <start>-<end>] #SOUPED_LINES <N> #SOUP_TRAILING_NEWLINE <0|1> 2. HEADER PARAMETERS: Path: Original absolute path EXACTLY, but JSON-escaped (e.g., ", \). #SOUP_PARTIAL_LINES: Optional. Use for localized edits. <start>-<end> are 1-indexed, inclusive original line numbers being replaced. : Exact integer count of content lines immediately following the header. <0|1>: 1 if the entire file (after edits) ends in \n, else 0. 3. LINE COUNTING LOGIC (): Empty content = 0. foo = 1. foo\n = 1. foo\nbar = 2. foo\nbar\n = 2. CRITICAL: Line counts must be mathematically exact. 4. OUTPUT RULES: FULL BLOCK: Replaces/creates the whole file. Omit #SOUP_PARTIAL_LINES. Output all N lines. PARTIAL BLOCK: Replaces ONLY lines <start> through <end>. Output the N replacement lines. The <0|1> trailing newline flag must reflect the FINAL whole file's state, not just the snippet. SPACING: Zero blank spacer lines between a header and its content, or between blocks.
